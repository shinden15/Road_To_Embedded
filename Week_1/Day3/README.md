## Background: SPI Protocol (read first, 30 min)

SPI (Serial Peripheral Interface) is a synchronous, full-duplex protocol with one master and one or more slaves.

**Four signals:**
- **SCLK** — clock generated by master
- **MOSI** — Master Out Slave In (master sends)
- **MISO** — Master In Slave Out (slave sends)
- **CS/SS** — Chip Select, active LOW — selects which slave is active

**Four modes (CPOL × CPHA):**

| Mode | CPOL | CPHA | Clock idle | Sample on |
|------|------|------|------------|-----------|
| 0 | 0 | 0 | LOW | Rising edge |
| 1 | 0 | 1 | LOW | Falling edge |
| 2 | 1 | 0 | HIGH | Falling edge |
| 3 | 1 | 1 | HIGH | Rising edge |

**How to find the mode for a device:** Look in the datasheet for CPOL/CPHA values, or look at the timing diagram — check what state the clock is in when idle and which edge data is sampled on.

**Before writing any code:** Open the datasheet for both the ST7920 and BMP280. Find:
- The SPI mode (CPOL/CPHA)
- The max clock frequency
- The CS polarity
- The bit order (MSB first or LSB first)

## Wiring

### BMP280 (SPI mode)

| BMP280 Pin | ESP32 Pin | Notes |
|------------|-----------|-------|
| VCC | 3.3V | 3.3V only |
| GND | GND | |
| SCL/SCK | GPIO14 | SPI clock |
| SDA/MOSI | GPIO13 | MOSI |
| SDO/MISO | GPIO12 | MISO |
| CSB/CS | GPIO15 | Chip select — pull LOW to enable SPI mode |

> **BMP280 SPI enable:** The CSB pin must be pulled LOW at power-on to enable SPI mode. Tie it LOW permanently for this exercise or control via GPIO.

### GME12864-11 / ST7920 (SPI mode)

| ST7920 Pin | ESP32 Pin | Notes |
|------------|-----------|-------|
| VCC | 5V | ST7920 needs 5V — use 5V rail from ESP32 devkit |
| GND | GND | |
| RS (CS) | GPIO5 | Chip select |
| R/W (MOSI) | GPIO23 | MOSI |
| E (SCLK) | GPIO18 | Clock |
| PSB | GND | Pull LOW to enable SPI mode (not parallel) |
| RST | 3.3V or GPIO | Pull HIGH or control via GPIO |
| BLA (backlight+) | 3.3V | Optional |
| BLK (backlight-) | GND | Optional |

> **ST7920 logic level:** The ST7920 uses 5V logic but is generally 3.3V tolerant on data lines. If you have issues, add a level shifter or use voltage dividers on MOSI and SCLK.

## Task 1 — ESP-IDF SPI Bus and Device Init (1 hr)

**Goal:** Configure the SPI bus and register devices correctly. ESP32 has 4 SPI peripherals: SPI0/SPI1 are used by flash, SPI2 (HSPI) and SPI3 (VSPI) are available for general use.

### SPI bus init (shared by all devices on the bus)

```c
#include "driver/spi_master.h"

#define SPI_HOST    SPI2_HOST   // HSPI
#define PIN_MOSI    GPIO_NUM_13
#define PIN_MISO    GPIO_NUM_12
#define PIN_SCLK    GPIO_NUM_14

void spi_bus_init(void) {
    spi_bus_config_t bus_cfg = {
        .mosi_io_num     = PIN_MOSI,
        .miso_io_num     = PIN_MISO,
        .sclk_io_num     = PIN_SCLK,
        .quadwp_io_num   = -1,   // not used
        .quadhd_io_num   = -1,   // not used
        .max_transfer_sz = 4096,
    };
    ESP_ERROR_CHECK(spi_bus_initialize(SPI_HOST, &bus_cfg, SPI_DMA_CH_AUTO));
}
```

### Device handle: BMP280

```c
#define BMP280_CS_PIN   GPIO_NUM_15
#define BMP280_CLK_HZ   1000000   // 1 MHz — conservative start

spi_device_handle_t bmp280_handle;

void bmp280_spi_init(void) {
    spi_device_interface_config_t dev_cfg = {
        .clock_speed_hz = BMP280_CLK_HZ,
        .mode           = 0,           // SPI mode 0 — verify in datasheet
        .spics_io_num   = BMP280_CS_PIN,
        .queue_size     = 1,
        .flags          = 0,
    };
    ESP_ERROR_CHECK(spi_bus_add_device(SPI_HOST, &dev_cfg, &bmp280_handle));
}
```

### Device handle: ST7920

```c
#define ST7920_CS_PIN   GPIO_NUM_5
#define ST7920_CLK_HZ   500000    // ST7920 max SPI clock is ~1MHz; start at 500kHz

spi_device_handle_t st7920_handle;

void st7920_spi_init(void) {
    spi_device_interface_config_t dev_cfg = {
        .clock_speed_hz = ST7920_CLK_HZ,
        .mode           = 3,           // ST7920 SPI uses mode 3 — verify in datasheet
        .spics_io_num   = ST7920_CS_PIN,
        .queue_size     = 4,
        .flags          = SPI_DEVICE_BIT_LSBFIRST,  // ST7920 sends LSB first — verify
    };
    ESP_ERROR_CHECK(spi_bus_add_device(SPI_HOST, &dev_cfg, &st7920_handle));
}
```

> **Check the datasheets** for both devices before hardcoding mode and bit order. The values above are starting points — verify against the timing diagrams.

## Task 2 — SPI Transfer Helpers (45 min)

**Goal:** Write reusable read/write functions for single bytes and bursts.

```c
// Write a single byte to a register
esp_err_t spi_write_reg(spi_device_handle_t dev,
                         uint8_t reg, uint8_t data) {
    spi_transaction_t t = {
        .length    = 16,                    // bits
        .tx_buffer = (uint8_t[]){reg & 0x7F, data},  // bit7=0 → write
        .rx_buffer = NULL,
    };
    return spi_device_transmit(dev, &t);
}

// Read a single byte from a register
esp_err_t spi_read_reg(spi_device_handle_t dev,
                        uint8_t reg, uint8_t *data) {
    uint8_t rx[2] = {0};
    spi_transaction_t t = {
        .length    = 16,
        .tx_buffer = (uint8_t[]){reg | 0x80, 0x00},  // bit7=1 → read
        .rx_buffer = rx,
    };
    esp_err_t err = spi_device_transmit(dev, &t);
    *data = rx[1];
    return err;
}

// Burst read: read multiple consecutive registers
esp_err_t spi_burst_read(spi_device_handle_t dev,
                          uint8_t reg, uint8_t *buf, size_t len) {
    uint8_t tx[len + 1];
    uint8_t rx[len + 1];
    memset(tx, 0, sizeof(tx));
    tx[0] = reg | 0x80;

    spi_transaction_t t = {
        .length    = (len + 1) * 8,
        .tx_buffer = tx,
        .rx_buffer = rx,
    };
    esp_err_t err = spi_device_transmit(dev, &t);
    memcpy(buf, rx + 1, len);
    return err;
}
```

> **BMP280 read/write protocol:** bit7 of the register address selects read (1) or write (0). Verify this in the BMP280 datasheet register map section.

## Task 3 — BMP280 Driver (2 hrs)

**Goal:** Read temperature and pressure from the BMP280 by implementing the driver from the datasheet — no library.

### Step 1: Verify communication

```c
void bmp280_verify(void) {
    uint8_t chip_id;
    spi_read_reg(bmp280_handle, 0xD0, &chip_id);  // register 0xD0 = chip_id
    // BMP280 chip_id = 0x58, BMP580 = 0x50
    ESP_LOGI(TAG, "BMP280 chip_id: 0x%02X (expect 0x58)", chip_id);
}
```

If you get `0xFF` or `0x00`: wiring issue or wrong SPI mode.

### Step 2: Read calibration data

BMP280 stores factory calibration coefficients in registers `0x88`–`0x9F`. These are needed to convert raw ADC values to real temperature and pressure.

```c
typedef struct {
    uint16_t dig_T1;
    int16_t  dig_T2;
    int16_t  dig_T3;
    uint16_t dig_P1;
    int16_t  dig_P2;
    int16_t  dig_P3;
    int16_t  dig_P4;
    int16_t  dig_P5;
    int16_t  dig_P6;
    int16_t  dig_P7;
    int16_t  dig_P8;
    int16_t  dig_P9;
} BMP280Calib;

void bmp280_read_calib(BMP280Calib *cal) {
    uint8_t raw[24];
    spi_burst_read(bmp280_handle, 0x88, raw, 24);

    cal->dig_T1 = (raw[1]  << 8) | raw[0];
    cal->dig_T2 = (raw[3]  << 8) | raw[2];
    cal->dig_T3 = (raw[5]  << 8) | raw[4];
    cal->dig_P1 = (raw[7]  << 8) | raw[6];
    cal->dig_P2 = (raw[9]  << 8) | raw[8];
    // ... continue for P3–P9
}
```

### Step 3: Configure and trigger measurement

```c
void bmp280_configure(void) {
    // Set oversampling and mode
    // osrs_t=2 (×2 oversampling temp), osrs_p=5 (×16 pressure), mode=3 (normal)
    spi_write_reg(bmp280_handle, 0xF4, (2 << 5) | (5 << 2) | 3);

    // Set standby time and filter
    // t_sb=5 (1000ms standby), filter=4
    spi_write_reg(bmp280_handle, 0xF5, (5 << 5) | (4 << 2));
}
```

### Step 4: Read and compensate raw data

```c
typedef struct { int32_t temp_cdeg; uint32_t press_pa; } BMP280Data;

BMP280Data bmp280_read(BMP280Calib *cal) {
    uint8_t raw[6];
    spi_burst_read(bmp280_handle, 0xF7, raw, 6);  // press MSB, LSB, XLSB, temp MSB, LSB, XLSB

    int32_t adc_P = ((int32_t)raw[0] << 12) | ((int32_t)raw[1] << 4) | (raw[2] >> 4);
    int32_t adc_T = ((int32_t)raw[3] << 12) | ((int32_t)raw[4] << 4) | (raw[5] >> 4);

    // Temperature compensation (from BMP280 datasheet section 4.2.3)
    int32_t var1 = ((((adc_T >> 3) - ((int32_t)cal->dig_T1 << 1)))
                    * cal->dig_T2) >> 11;
    int32_t var2 = (((((adc_T >> 4) - (int32_t)cal->dig_T1)
                    * ((adc_T >> 4) - (int32_t)cal->dig_T1)) >> 12)
                    * cal->dig_T3) >> 14;
    int32_t t_fine = var1 + var2;

    BMP280Data data;
    data.temp_cdeg = (t_fine * 5 + 128) >> 8;  // in 0.01°C units

    // Pressure compensation — implement from datasheet section 4.2.3
    // ...

    return data;
}
```

## Task 4 — ST7920 Display Driver (2.5 hrs)

**Goal:** Initialize the display and render text and graphics in SPI mode.

### ST7920 SPI protocol

The ST7920 uses a custom SPI framing:
- Each byte is sent as: **sync byte** (0xF8 for write) + **high nibble** (upper 4 bits, padded) + **low nibble** (lower 4 bits, padded)
- This is NOT standard SPI register addressing — it's the ST7920's serial protocol

```c
void st7920_send_byte(uint8_t is_data, uint8_t byte) {
    uint8_t sync  = is_data ? 0xFA : 0xF8;
    uint8_t high  = byte & 0xF0;
    uint8_t low   = (byte << 4) & 0xF0;

    uint8_t buf[3] = {sync, high, low};
    spi_transaction_t t = {
        .length    = 24,
        .tx_buffer = buf,
    };
    spi_device_transmit(st7920_handle, &t);
}

void st7920_cmd(uint8_t cmd)  { st7920_send_byte(0, cmd); }
void st7920_data(uint8_t d)   { st7920_send_byte(1, d);   }
```

### Initialization sequence

```c
void st7920_init(void) {
    vTaskDelay(pdMS_TO_TICKS(50));   // wait for VCC to stabilize

    st7920_cmd(0x30);   // function set: 8-bit, basic instruction
    vTaskDelay(pdMS_TO_TICKS(5));
    st7920_cmd(0x30);   // repeat
    vTaskDelay(pdMS_TO_TICKS(1));
    st7920_cmd(0x0C);   // display on, cursor off, blink off
    st7920_cmd(0x01);   // clear display
    vTaskDelay(pdMS_TO_TICKS(10));
    st7920_cmd(0x06);   // entry mode: increment, no shift

    ESP_LOGI(TAG, "ST7920 initialized");
}
```

### Write text (basic instruction mode)

ST7920 has a 4-line, 16-character text mode:

```c
static const uint8_t LINE_ADDR[] = {0x80, 0x90, 0x88, 0x98};  // DDRAM addresses

void st7920_set_cursor(uint8_t line, uint8_t col) {
    st7920_cmd(LINE_ADDR[line] + col);
}

void st7920_print(uint8_t line, uint8_t col, const char *str) {
    st7920_set_cursor(line, col);
    while (*str) {
        st7920_data(*str++);
    }
}
```

### Graphics mode — render a bitmap (stretch goal)

```c
void st7920_set_graphics_mode(void) {
    st7920_cmd(0x36);   // extended instruction set, graphics on
}

// Write a full 128×64 frame buffer
// ST7920 graphics RAM: 32 rows × 8 bytes per row (upper half), then lower half
void st7920_draw_frame(uint8_t *framebuf) {  // 1024 bytes
    st7920_set_graphics_mode();
    for (uint8_t y = 0; y < 32; y++) {
        st7920_cmd(0x80 | y);        // set vertical address
        st7920_cmd(0x80);            // set horizontal address to 0
        for (uint8_t x = 0; x < 16; x++) {
            st7920_data(framebuf[y * 16 + x]);
        }
    }
    for (uint8_t y = 0; y < 32; y++) {
        st7920_cmd(0x80 | y);
        st7920_cmd(0x88);            // horizontal address = 8 (right half)
        for (uint8_t x = 0; x < 16; x++) {
            st7920_data(framebuf[(y + 32) * 16 + x]);
        }
    }
}
```

### Display demo

```c
void display_task(void *arg) {
    char line[17];
    BMP280Calib cal;
    bmp280_read_calib(&cal);

    while (1) {
        BMP280Data data = bmp280_read(&cal);

        st7920_cmd(0x01);  // clear

        st7920_print(0, 0, "ESP32 Monitor");
        snprintf(line, sizeof(line), "Temp: %d.%02d C",
                 data.temp_cdeg / 100, data.temp_cdeg % 100);
        st7920_print(1, 0, line);
        snprintf(line, sizeof(line), "Press:%luhPa",
                 data.press_pa / 100);
        st7920_print(2, 0, line);

        vTaskDelay(pdMS_TO_TICKS(1000));
    }
}
```

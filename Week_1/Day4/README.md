## Background: I2C Protocol (read first, 20 min)

I2C (Inter-Integrated Circuit) is a two-wire, half-duplex, multi-device protocol.

**Two signals:**
- **SDA** — Serial Data Line (bidirectional)
- **SCL** — Serial Clock Line (generated by master)

**Key concepts:**
- **Addresses:** Every I2C device has a 7-bit address (plus R/W bit = 8 bits total). Multiple devices share the same bus — differentiated by address.
- **ACK/NACK:** After each byte, the receiver pulls SDA LOW (ACK) or releases it HIGH (NACK). NACK means "stop" or "error."
- **Clock stretching:** A slave can hold SCL LOW to pause the master while it's busy. Not all masters support this.
- **Pull-up resistors:** SDA and SCL are open-drain — they need pull-up resistors (typically 4.7kΩ) to VCC. Many breakout boards include these; check before adding more.

**Typical read transaction:**
```
START → [addr + W] → [reg] → RESTART → [addr + R] → [data byte] → STOP
```

**Before writing any code:** Open the datasheet for both the MPU6050 and DS3231. Find:
- The I2C address (and whether there's an address pin to change it)
- The WHO_AM_I or similar identity register
- The data registers for the values you want to read

## Wiring

### MPU6050

| MPU6050 Pin | ESP32 Pin | Notes |
|-------------|-----------|-------|
| VCC | 3.3V | 3.3V operation |
| GND | GND | |
| SDA | GPIO21 | ESP32 default I2C SDA |
| SCL | GPIO22 | ESP32 default I2C SCL |
| AD0 | GND | I2C address = 0x68 (HIGH = 0x69) |
| INT | GPIO4 | Interrupt output (optional for now) |

### DS3231

| DS3231 Pin | ESP32 Pin | Notes |
|------------|-----------|-------|
| VCC | 3.3V | |
| GND | GND | |
| SDA | GPIO21 | Same I2C bus as MPU6050 |
| SCL | GPIO22 | Same I2C bus as MPU6050 |
| SQW | (optional) | Square wave / interrupt output |

> Both devices can share the same I2C bus — they have different addresses (MPU6050 = 0x68, DS3231 = 0x68... wait — **check both datasheets**. DS3231 is 0x68 too. This is a problem — verify and use `AD0=HIGH` on MPU6050 to set it to 0x69).

## Task 1 — I2C Bus Init and Scanner (45 min)

**Goal:** Initialize the I2C master and write a bus scanner to confirm devices are detected.

### I2C master init

```c
#include "driver/i2c.h"

#define I2C_PORT    I2C_NUM_0
#define I2C_SDA     GPIO_NUM_21
#define I2C_SCL     GPIO_NUM_22
#define I2C_FREQ_HZ 400000   // 400kHz fast mode

void i2c_master_init(void) {
    i2c_config_t cfg = {
        .mode             = I2C_MODE_MASTER,
        .sda_io_num       = I2C_SDA,
        .scl_io_num       = I2C_SCL,
        .sda_pullup_en    = GPIO_PULLUP_ENABLE,
        .scl_pullup_en    = GPIO_PULLUP_ENABLE,
        .master.clk_speed = I2C_FREQ_HZ,
    };
    i2c_param_config(I2C_PORT, &cfg);
    i2c_driver_install(I2C_PORT, cfg.mode, 0, 0, 0);
}
```

**Note on pull-ups:** `GPIO_PULLUP_ENABLE` enables the ESP32's internal pull-ups (~45kΩ). These are weak — if you have long wires or many devices, add external 4.7kΩ pull-ups from SDA/SCL to 3.3V. For this session, internal pull-ups should be sufficient.

### I2C bus scanner

```c
void i2c_scan(void) {
    ESP_LOGI(TAG, "Scanning I2C bus...");
    for (uint8_t addr = 1; addr < 127; addr++) {
        i2c_cmd_handle_t cmd = i2c_cmd_link_create();
        i2c_master_start(cmd);
        i2c_master_write_byte(cmd, (addr << 1) | I2C_MASTER_WRITE, true);
        i2c_master_stop(cmd);

        esp_err_t ret = i2c_master_cmd_begin(I2C_PORT, cmd, pdMS_TO_TICKS(10));
        i2c_cmd_link_delete(cmd);

        if (ret == ESP_OK) {
            ESP_LOGI(TAG, "Found device at 0x%02X", addr);
        }
    }
    ESP_LOGI(TAG, "Scan complete");
}
```

**Expected output:**
```
Found device at 0x68  (MPU6050 with AD0=LOW, or DS3231)
Found device at 0x69  (MPU6050 with AD0=HIGH)
```

Run the scanner first — do not proceed until both devices are detected.

## Task 2 — I2C Read/Write Helpers (30 min)

**Goal:** Write clean register read/write functions used by both drivers.

```c
esp_err_t i2c_write_reg(uint8_t dev_addr, uint8_t reg, uint8_t data) {
    i2c_cmd_handle_t cmd = i2c_cmd_link_create();
    i2c_master_start(cmd);
    i2c_master_write_byte(cmd, (dev_addr << 1) | I2C_MASTER_WRITE, true);
    i2c_master_write_byte(cmd, reg,  true);
    i2c_master_write_byte(cmd, data, true);
    i2c_master_stop(cmd);
    esp_err_t ret = i2c_master_cmd_begin(I2C_PORT, cmd, pdMS_TO_TICKS(100));
    i2c_cmd_link_delete(cmd);
    return ret;
}

esp_err_t i2c_read_reg(uint8_t dev_addr, uint8_t reg, uint8_t *data) {
    i2c_cmd_handle_t cmd = i2c_cmd_link_create();
    i2c_master_start(cmd);
    i2c_master_write_byte(cmd, (dev_addr << 1) | I2C_MASTER_WRITE, true);
    i2c_master_write_byte(cmd, reg, true);
    i2c_master_start(cmd);   // repeated START
    i2c_master_write_byte(cmd, (dev_addr << 1) | I2C_MASTER_READ, true);
    i2c_master_read_byte(cmd, data, I2C_MASTER_NACK);
    i2c_master_stop(cmd);
    esp_err_t ret = i2c_master_cmd_begin(I2C_PORT, cmd, pdMS_TO_TICKS(100));
    i2c_cmd_link_delete(cmd);
    return ret;
}

esp_err_t i2c_burst_read(uint8_t dev_addr, uint8_t reg,
                          uint8_t *buf, size_t len) {
    i2c_cmd_handle_t cmd = i2c_cmd_link_create();
    i2c_master_start(cmd);
    i2c_master_write_byte(cmd, (dev_addr << 1) | I2C_MASTER_WRITE, true);
    i2c_master_write_byte(cmd, reg, true);
    i2c_master_start(cmd);
    i2c_master_write_byte(cmd, (dev_addr << 1) | I2C_MASTER_READ, true);
    i2c_master_read(cmd, buf, len, I2C_MASTER_LAST_NACK);
    i2c_master_stop(cmd);
    esp_err_t ret = i2c_master_cmd_begin(I2C_PORT, cmd, pdMS_TO_TICKS(100));
    i2c_cmd_link_delete(cmd);
    return ret;
}
```

**Understand every call:**
- Why is there a repeated START before the read phase?
- What is the difference between `I2C_MASTER_NACK` and `I2C_MASTER_LAST_NACK`?
- What does `true` mean in `i2c_master_write_byte(..., true)`? (ACK check)

## Task 3 — MPU6050 Driver (2.5 hrs)

**Goal:** Read accelerometer and gyroscope data from raw registers. No MPU6050 library.

### MPU6050 I2C addresses

```c
#define MPU6050_ADDR      0x68   // AD0=LOW
#define MPU6050_ADDR_ALT  0x69   // AD0=HIGH
```

### Step 1: Wake up the device

MPU6050 starts in sleep mode. You must clear the sleep bit:

```c
// Register 0x6B = PWR_MGMT_1
// Bit 6 = SLEEP — clear it to wake up
// Set CLKSEL=1 (PLL with X gyro) for better clock accuracy
void mpu6050_wake(void) {
    i2c_write_reg(MPU6050_ADDR, 0x6B, 0x01);  // SLEEP=0, CLKSEL=1
    vTaskDelay(pdMS_TO_TICKS(100));
}
```

### Step 2: Verify WHO_AM_I

```c
void mpu6050_verify(void) {
    uint8_t who_am_i;
    i2c_read_reg(MPU6050_ADDR, 0x75, &who_am_i);
    // MPU6050 WHO_AM_I = 0x68
    ESP_LOGI(TAG, "MPU6050 WHO_AM_I: 0x%02X (expect 0x68)", who_am_i);
}
```

### Step 3: Configure full-scale range

```c
// Register 0x1C = ACCEL_CONFIG
// Bits 4:3 = AFS_SEL: 00=±2g, 01=±4g, 10=±8g, 11=±16g
void mpu6050_set_accel_range(uint8_t range) {
    i2c_write_reg(MPU6050_ADDR, 0x1C, (range & 0x3) << 3);
}

// Register 0x1B = GYRO_CONFIG
// Bits 4:3 = FS_SEL: 00=±250°/s, 01=±500°/s, 10=±1000°/s, 11=±2000°/s
void mpu6050_set_gyro_range(uint8_t range) {
    i2c_write_reg(MPU6050_ADDR, 0x1B, (range & 0x3) << 3);
}
```

### Step 4: Read raw data

```c
typedef struct {
    int16_t ax, ay, az;   // raw accelerometer counts
    int16_t temp_raw;     // raw temperature
    int16_t gx, gy, gz;   // raw gyroscope counts
} MPU6050Raw;

void mpu6050_read_raw(MPU6050Raw *raw) {
    uint8_t buf[14];
    // Registers 0x3B–0x48: ACCEL_XOUT_H through GYRO_ZOUT_L
    i2c_burst_read(MPU6050_ADDR, 0x3B, buf, 14);

    raw->ax       = (int16_t)((buf[0]  << 8) | buf[1]);
    raw->ay       = (int16_t)((buf[2]  << 8) | buf[3]);
    raw->az       = (int16_t)((buf[4]  << 8) | buf[5]);
    raw->temp_raw = (int16_t)((buf[6]  << 8) | buf[7]);
    raw->gx       = (int16_t)((buf[8]  << 8) | buf[9]);
    raw->gy       = (int16_t)((buf[10] << 8) | buf[11]);
    raw->gz       = (int16_t)((buf[12] << 8) | buf[13]);
}
```

### Step 5: Convert to real units

```c
typedef struct {
    float ax_g, ay_g, az_g;   // acceleration in g
    float temp_c;              // temperature in °C
    float gx_dps, gy_dps, gz_dps;  // angular velocity in °/s
} MPU6050Data;

// Scale factors depend on configured range
// ±2g  → 16384 LSB/g
// ±250°/s → 131 LSB/(°/s)
#define ACCEL_SCALE  16384.0f
#define GYRO_SCALE   131.0f

MPU6050Data mpu6050_convert(MPU6050Raw *raw) {
    MPU6050Data d;
    d.ax_g   = raw->ax / ACCEL_SCALE;
    d.ay_g   = raw->ay / ACCEL_SCALE;
    d.az_g   = raw->az / ACCEL_SCALE;
    d.temp_c = raw->temp_raw / 340.0f + 36.53f;  // from datasheet
    d.gx_dps = raw->gx / GYRO_SCALE;
    d.gy_dps = raw->gy / GYRO_SCALE;
    d.gz_dps = raw->gz / GYRO_SCALE;
    return d;
}
```

**Sanity check:** With the sensor flat and still:
- `az_g` ≈ ±1.0 (gravity)
- `ax_g`, `ay_g` ≈ 0
- `gx_dps`, `gy_dps`, `gz_dps` ≈ 0 (with slight noise)
- `temp_c` ≈ your room temperature

## Task 4 — DS3231 RTC Driver (2 hrs)

**Goal:** Read and set the real-time clock. DS3231 uses BCD encoding — understand and implement the conversion.

### DS3231 address

```c
#define DS3231_ADDR  0x68
```

> Since MPU6050 at 0x68 and DS3231 at 0x68 conflict, wire MPU6050 `AD0=HIGH` (to 3.3V) to use address 0x69.

### BCD encoding

DS3231 stores time in BCD (Binary Coded Decimal):
- `0x59` = 59 (seconds)
- `0x23` = 23 (hours)
- Upper nibble = tens digit, lower nibble = units digit

```c
static uint8_t bcd_to_dec(uint8_t bcd) {
    return (bcd >> 4) * 10 + (bcd & 0x0F);
}

static uint8_t dec_to_bcd(uint8_t dec) {
    return ((dec / 10) << 4) | (dec % 10);
}
```

### DS3231 register map (from datasheet)

| Register | Content |
|----------|---------|
| 0x00 | Seconds (BCD, 0–59) |
| 0x01 | Minutes (BCD, 0–59) |
| 0x02 | Hours (BCD, bit6=12/24hr mode) |
| 0x03 | Day of week (1–7) |
| 0x04 | Date (BCD, 1–31) |
| 0x05 | Month/century (BCD, bit7=century) |
| 0x06 | Year (BCD, 0–99) |
| 0x11 | Temperature MSB (signed, °C) |
| 0x12 | Temperature LSB (bits 7:6, 0.25°C resolution) |

### Time struct and read function

```c
typedef struct {
    uint8_t seconds;
    uint8_t minutes;
    uint8_t hours;
    uint8_t day;
    uint8_t date;
    uint8_t month;
    uint8_t year;   // 2-digit year (add 2000)
} DS3231Time;

void ds3231_read_time(DS3231Time *t) {
    uint8_t buf[7];
    i2c_burst_read(DS3231_ADDR, 0x00, buf, 7);

    t->seconds = bcd_to_dec(buf[0] & 0x7F);
    t->minutes = bcd_to_dec(buf[1] & 0x7F);
    t->hours   = bcd_to_dec(buf[2] & 0x3F);   // assume 24hr mode
    t->day     = bcd_to_dec(buf[3] & 0x07);
    t->date    = bcd_to_dec(buf[4] & 0x3F);
    t->month   = bcd_to_dec(buf[5] & 0x1F);
    t->year    = bcd_to_dec(buf[6]);
}
```

### Set the time

```c
void ds3231_set_time(DS3231Time *t) {
    uint8_t buf[7] = {
        dec_to_bcd(t->seconds),
        dec_to_bcd(t->minutes),
        dec_to_bcd(t->hours),
        dec_to_bcd(t->day),
        dec_to_bcd(t->date),
        dec_to_bcd(t->month),
        dec_to_bcd(t->year),
    };
    // Write 7 bytes starting at register 0x00
    i2c_cmd_handle_t cmd = i2c_cmd_link_create();
    i2c_master_start(cmd);
    i2c_master_write_byte(cmd, (DS3231_ADDR << 1) | I2C_MASTER_WRITE, true);
    i2c_master_write_byte(cmd, 0x00, true);
    i2c_master_write(cmd, buf, 7, true);
    i2c_master_stop(cmd);
    i2c_master_cmd_begin(I2C_PORT, cmd, pdMS_TO_TICKS(100));
    i2c_cmd_link_delete(cmd);
}
```

### Read DS3231 temperature

```c
float ds3231_read_temp(void) {
    uint8_t msb, lsb;
    i2c_read_reg(DS3231_ADDR, 0x11, &msb);
    i2c_read_reg(DS3231_ADDR, 0x12, &lsb);
    return (int8_t)msb + ((lsb >> 6) * 0.25f);
}
```

## Task 5 — Display on ST7920 and Serial (1 hr)

**Goal:** Show live MPU6050 and DS3231 data on the display and over UART.

```c
void sensor_task(void *arg) {
    DS3231Time t = {0, 30, 14, 1, 20, 4, 25};  // set initial time: 14:30:00, 20 Apr 2025
    ds3231_set_time(&t);

    mpu6050_wake();
    mpu6050_set_accel_range(0);  // ±2g
    mpu6050_set_gyro_range(0);   // ±250°/s

    char line[17];
    while (1) {
        MPU6050Raw raw;
        mpu6050_read_raw(&raw);
        MPU6050Data imu = mpu6050_convert(&raw);

        ds3231_read_time(&t);
        float rtc_temp = ds3231_read_temp();

        // Serial output
        ESP_LOGI(TAG, "Time: 20%02d-%02d-%02d %02d:%02d:%02d",
                 t.year, t.month, t.date, t.hours, t.minutes, t.seconds);
        ESP_LOGI(TAG, "Accel: %.2fg %.2fg %.2fg",
                 imu.ax_g, imu.ay_g, imu.az_g);
        ESP_LOGI(TAG, "Gyro:  %.1f %.1f %.1f dps",
                 imu.gx_dps, imu.gy_dps, imu.gz_dps);
        ESP_LOGI(TAG, "RTC temp: %.2f C", rtc_temp);

        vTaskDelay(pdMS_TO_TICKS(500));
    }
}
```
